🔑 Core Concepts

The eight terms most important for everyday work with Claude.

Haiku / Sonnet / Opus / Fable (Module 0)

The four model tiers: Haiku (the fastest, most cost-efficient model), Sonnet (the all-rounder for everyday work), Opus (the flagship model for maximum analytical depth), Fable (Mythos class, positioned above Opus since June 2026 — for exceptional cases only).

Projects (Module 0)

A workspace that bundles files, instructions and chat history around one topic — set it up once, never brief Claude again; applies to every chat within the Project.

Connector (Module 0, Module 3)

An interface to an external service (Gmail, Google Drive, Calendar, Slack, Teams, Outlook). The access granted is significant — a connected Gmail account means the AI can read the entire mailbox.

"Lethal Trifecta" (Module 3)

Security concept: things become dangerous when an AI has (1) access to private data, (2) exposure to untrusted external content, and (3) the ability to send information out — all at the same time.

Rolling 5-Hour Window (Module 9)

Usage limits run on a rolling 5-hour window — spreading work across several sessions in a day lets older usage "roll off" before a new session starts.

Context Window / Chat Discipline (Module 1)

Claude re-reads the entire chat history with every reply. The longer and more mixed in topic a chat becomes, the less accurate and slower the answers get — a fresh chat per topic keeps context clean.

Skill (Module 4, Module 5)

A reusable, saved workflow that Claude can call on when needed instead of being instructed from scratch every time — the difference between a one-off instruction and a permanent tool.

Claude Cowork (Module 0, Module 6)

An autonomous desktop agent with access to authorised local folders — it reads, creates and edits files independently. Runs via the desktop app (Mac/Windows), included on every paid plan; web/mobile are currently in beta rollout for remote control.

📎 Other Terms

Listed alphabetically.

Adaptive Thinking (Module 0)

Sonnet 5, Opus 4.8 and Fable 5 automatically decide how much time to spend on an answer — instant for simple questions, longer in the background for complex analyses. Only Haiku 4.5 deliberately skips this.

Chain of Thought (Module 2)

Prompting technique: Claude is instructed to "think step by step" before giving its answer.

Claude Tag (@Claude in Slack) (Module 8)

Mentioning @Claude in Slack channels and threads delegates tasks to it; Claude then continues working asynchronously in the background.

Few-Shot Prompting (Module 2)

One to three concrete examples often show the desired result more clearly than a long description.

Incognito Mode (Module 1)

Chats are not permanently saved in the sidebar (ghost icon) — ideal for private or one-off questions.

Memory (Module 1)

Using the command "Please create a memory", Claude remembers facts about you that it automatically takes into account in future chats.

NotebookLM (Module 4)

A Google tool for gathering knowledge from large sets of documents (including "Deep Research"), often used before building a Skill.

Personal Instructions (Module 6)

Permanently saved instructions (Profile → "Instructions for Claude") that adjust Claude's default behaviour across every chat.

Plugin (Module 0)

Bundles several Skills together into one package.

Prompt Caching (Module 2)

Saves time, since Claude doesn't need to re-read static documents.

Shared Context (Module 7)

Since March 2026: the Excel and PowerPoint Add-ins share a common context — data from an open Excel sheet can be used directly in a PowerPoint slide without copy-paste.

System Prompt (Module 2)

A top-level instruction that sets Claude's behaviour for an entire conversation or a Project.

Token (Module 9)

The unit usage limits are measured in; usage grows with the length of a chat, since every new message re-reads the entire prior conversation.

VBA Macro (Module 7)

Claude can write VBA code for Excel but cannot execute it itself — starting it remains a manual action.

Web Search (Module 1)

Claude's ability to retrieve current information from the internet instead of relying solely on training knowledge.

XML Tags (Module 2)

A structuring aid for prompts (e.g. <instructions>, <context>) used to clearly separate different parts of a prompt.
