@Claude in Slack — Claude Tag

Claude Tag brings agentic AI directly into your team: tag @Claude, delegate tasks, build context across channels — while Claude works asynchronously in the background.

20 min. learning time 5 quiz questions Includes video

What is Claude Tag?

Claude Tag is a new way for teams to work with Claude. Instead of one person chatting privately with an AI, Claude joins a Slack channel as a full team member — with persistent context, access to approved tools, and the ability to handle tasks asynchronously over hours or even days.

Real-world numbers

At Anthropic itself, 65% of the product team's code is already created by their internal version of Claude Tag. The pattern is spreading well beyond engineering — to data analysis, support tickets and bug investigation.

1. From Chat to Team Member — The Conceptual Shift

All previous Claude tools are designed for one person: one chat, one user, one task. Claude Tag breaks this pattern:

FeatureClaude ChatClaude Tag
UsersIndividual, privateWhole team, visible in channel
ContextPer session, no memoryPersistent, grows over time
Working styleSynchronous, waits for responseAsynchronous, works in background
InitiativeOnly on requestTakes initiative (Ambient Mode)

2. How @Claude Works in Slack

The basic principle is simple — like mentioning a colleague:

  1. Type @Claude followed by your task in any channel
  2. Claude breaks the task into stages and works through them using available tools
  3. Claude responds in the thread when the task is done — visible to everyone
Example prompts for IEG

@Claude Create a summary of all open action items from the last 5 meeting notes in the #deal-team channel, grouped by owner.

@Claude Search all messages from the last week in #research and create a list of all mentioned companies with their key metrics.

3. The Four Core Advantages

@Claude is multiplayer

Within a given Slack channel, there is one Claude that interacts with everyone. Anyone can see what it is working on and pick up the conversation from where the last colleague left off — just like with a real team member.

@Claude learns over time

Claude builds context by following along with its channel — no re-briefing needed. Claude can even automatically learn from other Slack channels and data sources if granted permission. Note: private channels are never read.

@Claude takes initiative (Ambient Mode)

With ambient behaviour enabled, Claude proactively keeps you updated: it flags relevant information from across connected channels and tools, and follows up on threads or tasks that have gone quiet — without explicit tagging.

@Claude works asynchronously

Set a task and move on. Claude can schedule tasks for itself and pursue projects autonomously over hours or days. Multiple Claudes can be delegated in parallel — the pace of work multiplies.

4. Getting Started in 4 Steps

  1. Pair Claude Tag with your Slack workspace (Admin settings → Claude in Slack)
  2. Give Claude access to relevant tools (e.g. Google Drive, Notion, codebase)
  3. Set a monthly token spend limit for the organisation
  4. Test Claude in a private channel to confirm everything works
Migration

Claude Tag replaces the existing "Claude in Slack" app. The migration happens automatically on 3 August 2026 — admins can migrate earlier on their own timeline.

5. Access Control and Security

Claude Tag is designed from the ground up for teams and organisations:

  • Channel-specific identities: A Claude for sales has no access to engineering data and vice versa — context and memories stay strictly scoped to configured channels.
  • Three permission levels: Organisation → Workspace → Individual channel. Each level inherits the permissions of the level above.
  • Token limits: Admins set monthly budgets per organisation and per channel. Tasks that would exceed a limit are declined — never silently cut short.
  • Full audit log: Every action Claude takes is logged — including who made the request. In Slack, posts appear from the Claude app with a link back to the originating thread.
Important for IEG

Claude Tag is in beta for Claude Enterprise and Team customers. For confidential deal data, always use the IEG Enterprise Account and carefully configure which channels and tools are accessible. Explicitly exclude confidential channels.

6. Direct Messages and Conversation Separation

Beyond channel tagging, Claude can also be sent direct messages — it responds privately using the personal tools and connectors you have set up. Important: Slack conversations and Claude web chats are strictly separate and not mutually visible.

Exercise 1

3 @Claude prompts for the IEG day-to-day

Claude Tag is most valuable for recurring, cross-team tasks. Develop three concrete prompts.

  • Which tasks in your daily work are recurring and rule-based?
  • Which information is regularly shared between channels?
  • Which tasks could run asynchronously in the background?
  • Write 3 complete @Claude prompts — with a specific channel, task and desired output format.
Template:

@Claude [Task] from [Source/Channel]. Format: [Table / List / Summary]. By [Time / immediately].

Goal: Concrete, immediately actionable prompts — not abstract ideas.
Examples
  • Weekly pipeline update: @Claude Every Friday at 4:30pm, create a structured pipeline summary from #deal-updates: open deals by stage, new targets this week, next steps. Format: table.
  • Research digest: @Claude Search #research and #market-news from the last 7 days and create a list of the 5 most important M&A trends with a 2-sentence summary each.
  • Action item tracker: @Claude Read the meeting notes posted today in #meetings and create a table of all action items with owner and deadline.
