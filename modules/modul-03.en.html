Safe Handling of Data and Tools

Fundamentals for using Claude safely: sensitive content, typical risk areas, connectors and key rules for privacy-conscious working.

15 min. learning time 7 quiz questions

1. What Must Never Go into a Personal Account

Some data should never be entered into a personal AI account. If this has already happened: simply stop from now on.

Never paste
  • Source code or content from the company codebase
  • Client or customer data: names, emails, phone numbers, addresses, health information
  • Unpublished plans, roadmaps, designs or prototypes
  • Non-public financial data: revenues, margins, budgets, forecasts, deal terms
  • Anything marked "confidential", "internal" or under NDA
  • Access credentials, passwords, API keys, security settings
  • Complete contracts or legal documents with real names and terms
  • Recordings or minutes of internal meetings

Anyone needing AI support for this content should first anonymise or use the IEG Claude Account.

2. The Side-Project Trap

Using your employer's AI account to develop a private side project? In the worst case, the project belongs to the employer. The simple rule: personal work on personal devices, with the personal account.

3. Connectors: Useful and Risky

A connector links the AI to another service: Gmail, Google Drive, Calendar, Slack, Teams, Outlook. The access is enormous — connecting Gmail means the AI can read the entire inbox.

A connected AI can be controlled through manipulated content in emails or documents — known as the "Lethal Trifecta".

The Lethal Trifecta

AI has access to private data + reads external content it cannot control + can send information outward = attackers can steal data.

4. Using Connectors Safely

  • Never connect work accounts to a personal AI account.
  • Connect as little as possible, with minimal access.
  • Only use official connectors.
  • Review monthly and disconnect unused connections.
Conclusion

Connectors belong in the IEG Claude Account — where security is professionally guaranteed.

Practical exercise

Apply what you've learned

Assess inputs by risk and practice rephrasing problematic inputs safely.

Exercise 1

Assess input risk

Not everything you could type into an AI chat should be typed into it. Sort the following examples into three risk categories.

  • Create a table with three columns: Uncritical, Only anonymised, Not in personal account.
  • Sort these inputs: (a) General industry research, (b) Meeting notes with client names, (c) Internal deal memo with valuation figures, (d) Publicly available press release, (e) Excel file with client revenues, (f) Generic email template without names.
  • Justify each input in one sentence, explaining why it belongs in that category.
Example prompt:

I'll give you 6 different inputs that an M&A analyst might type into an AI chat. Sort them into three categories: uncritical, use only anonymised, not in personal account. Justify each classification in one sentence. Format: table.

Goal: Develop practical data sensitivity — recognise what may be entered where.

Exercise 2

Anonymise sensitive data

Practice making a real document safe for Claude before uploading.

  • Take a document that contains sensitive data (e.g. a contract or financial table).
  • Replace all real names with placeholders: "Company A", "Mr Smith" → "Contact Person 1".
  • Replace real figures with approximate ranges: "€4.2m revenue" → "mid single-digit million revenue".
  • Upload the anonymised version to Claude and ask your question.
Example prompt:

Analyse the attached contract for unusual clauses. Company names and figures have been anonymised. Focus on: payment terms, liability limitations and termination rights.

Goal: Anonymisation in 5 minutes — then use Claude safely for any sensitive document.
