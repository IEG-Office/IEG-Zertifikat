Prompting and Structured Instructions

The six building blocks of a strong prompt, XML tags, Few-Shot, Chain of Thought, Prompt Chaining, Self-Reflection, Pre-filling and System Prompts — with concrete examples from the M&A day-to-day.

35 min. learning time 10 quiz questions Includes video

1. Prompt Engineering as an Iterative Science

Prompt engineering is not a one-time exercise — it is an empirical science. You start with a base instruction, analyse the AI's errors and expand the prompt step by step until the result is perfect. The goal: structure clear instructions and context so the AI delivers the desired result on the first attempt ("one-shot").

2. The 6 Building Blocks of the Perfect Prompt

BlockDescriptionExample
RoleAssign Claude a professional identity"You are an experienced social media consultant"
TaskThe core — always start with a verb"Analyse", "Create", "Compare"
ContextRelevant background informationTarget audience, problem statement, tone
ProcessLogical individual steps (roadmap)"Step 1: Research, Step 2: Analysis"
Stop criteriaBoundaries against hallucinations"If unsure, say so openly"
OutputDefine the exact formatTable, list, JSON, word count
Why stop criteria?

Without clear boundaries, Claude will guess rather than admit uncertainty. Stop criteria such as "do not use technical jargon" prevent hallucinations.

3. From Weak to Excellent — A Comparison

The same information need, three quality levels. The example comes from the M&A day-to-day: you need an initial profile of a potential target.

❌ Weak prompt

"Tell me something about company XY."
Problem: no role, no structure, no format, no boundaries. Claude guesses what is meant, delivers arbitrary prose and may invent figures.

➖ Good prompt

"You are an M&A analyst. Create a short company profile for [Company XY]: business model, market position and possible acquisition rationale. Respond as a structured list."
Better: role, task and output format defined. But stop criteria and a clear scope are missing.

✅ Excellent prompt

"You are an experienced M&A analyst. Create a structured company profile for [Company XY] with exactly these sections: (1) Business model & revenue streams, (2) Market position & two to three competitors, (3) Possible acquisition rationale from a buyer perspective. Format: table plus one explanatory sentence each. Maximum 200 words. If information is not publicly available, mark it explicitly as 'not publicly available' — no estimates, no invented figures."

4. Structuring with XML Tags

Anthropic recommends XML tags (e.g. <instructions>, <context>, <example>) to cleanly separate information. Claude was trained on this structure and can reference individual sections specifically. Output control: place the final result in tags such as <verdict> for automated further processing.

5. Advanced Prompting Techniques

Few-Shot Prompting

Show Claude 1 to 3 examples — often more powerful than any long description.

Chain of Thought

Instruct Claude to "think step by step" — massively improves logical reasoning.

Prompt Chaining

Break a large task into small, sequential prompts. Check each intermediate result.

Self-Reflection

Ask Claude to review its own answer for errors and revise it.

Pre-filling

Provide the start of the answer (e.g. { for JSON) — strictly enforces the format.

6. Efficiency through System Prompts & Caching

  • System prompts: Offload static information (e.g. company forms).
  • Prompt caching: Saves time as Claude does not need to re-read static documents with every request.

7. The Order of Analysis

Always form before sketch — clear facts first, then interpretive data.

Remember

Structured data provides the context in which unstructured data is correctly interpreted.

Exercise 1

Professionally rebuild a weak prompt

A vague prompt delivers vague results. Systematically rework the following prompt using the 6 building blocks from this module.

  • Starting prompt: "Write me something about a company."
  • Add: Role (who should Claude be?), Task (concrete verb), Context (industry, purpose), Process (sequence of steps), Stop criteria (no guessing) and Output format (table, prose, word limit).
  • Test both versions in Claude and compare the quality of results.
  • Document the three most important differences.
Example prompt (after reworking):

You are an M&A analyst. Create a short profile of a mid-sized company in industrial automation. Structure: (1) Business model in 2 sentences, (2) estimated revenue size, (3) 3 relevant competitors. Max. 150 words. If data is unavailable, mark it explicitly.

Goal: Improve prompt quality through systematic structure — from vague input to precise output.

Exercise 2

Prompt Chaining for analysis tasks

Complex tasks become manageable when broken into multiple sequential prompts. Practise this with a typical analysis.

  • Prompt 1: Create a short profile for a company of your choice (business model, market position, key figures).
  • Prompt 2: Do the same for two further companies in the same industry.
  • Prompt 3: Summarise all three profiles in a comparison table, sorted by strategic fit.
  • Prompt 4: Create a management summary (max. 100 words) of the result.
  • Check after each step: is the intermediate result correct and complete?
Example prompt (Step 3):

Summarise the profiles of Companies A, B and C in a comparison table. Columns: Company, Revenue, Employees, Market position, Strategic fit (1–5 with rationale). Sort by fit descending.

Goal: Make complex tasks manageable through Prompt Chaining — every intermediate result is verifiable.
Approach
  • Step 1–2: Each company profile should contain: business model (2–3 sentences), estimated revenue range, 2–3 competitors, main strengths.
  • Step 3: The table makes differences immediately visible — sort by strategic fit to identify the best candidate.
  • Step 4: The management summary should name the top candidate with one clear reason.
  • Key insight: Checking each step prevents errors from carrying over into subsequent prompts.
