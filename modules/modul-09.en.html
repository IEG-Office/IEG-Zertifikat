Managing Usage Limits: Working Efficiently with Claude

How token costs work, and 23 practical habits to stop hitting usage limits — covering file uploads, model choice, and session planning.

20 min. learning time 6 quiz questions

1. Why Usage Limits Happen: How Token Costs Work

Claude measures usage in tokens — roughly comparable to a word. On every new message, Claude re-reads the entire conversation history from the start: every prior message and every prior answer. That means the first message in a chat is cheap, but every following message gets more expensive because more history has to be re-read.

The numbers behind it

One user tracked their usage and found that roughly 98.5% of tokens were spent purely re-reading conversation history — only 1.5% went toward the actual output. A 20-message session was estimated at roughly 105,000 tokens, and a 30-message session at roughly 232,000 tokens.

The common thread behind every habit below: avoid wasting tokens on repeated context, so they're available for the actual task.

2. Habits You Might Not Know About

#HabitWhy it saves tokens
1Convert files before uploadingA PDF page costs roughly 1,500–3,000 tokens, a 1000×1000 screenshot roughly 1,300 tokens. Copy the relevant text into a plain text/Markdown file first (tip: use doc.new, paste the text, download as .md).
2Plan in Chat, build in CoworkAnthropic has confirmed that file creation (spreadsheets, docs, presentations) uses more of the usage limit than regular chat messages. Nail the structure and assumptions in Chat first, then ask Cowork to "build this exact file."
3Say "ask me questions" instead of a long promptA long prompt gets re-counted every time the conversation is re-read. Short prompt + AskUserQuestion tool: clarifying questions are generated once, answers via clicks are cheap.
4Speak answers instead of typing themTyped follow-ups tend to be vague ("make it better") and trigger more messages. Spoken answers tend to carry more context in one shot — fewer messages, fewer context reloads.
5Stop asking for full redosInstead of "redo the report," name the specific section that's wrong. A full redo regenerates the entire output. Bonus: adding "No commentary, just the output" cuts down on verbose preambles.
6Batch tasks into one messageThree separate prompts = three full context reloads. One prompt with three tasks = one reload.
7Reuse the same prompt structureAnthropic has confirmed that similar, frequently-used prompts get partially cached (exact mechanism not published). A stable prompt template with only the variable part swapped can help.
8Edit your message instead of sending a follow-up (in Chat)Chat lets you edit and regenerate a previous message — the old exchange gets replaced, not stacked on top.
9Pick the right product/model for the taskRough mapping: quick question → Chat with Haiku; report from files → Cowork with Opus; chart from data → Code with Sonnet. Each product carries different token costs per interaction; Chat's "Research" feature is thorough but token-intensive.

3. The Basics That Still Matter

#HabitWhy it saves tokens
10Keep "About Me"/context files under ~2,000 wordsCowork re-reads the working folder before every single task. Tip: at session end, ask for a session-notes.md with key decisions/next steps, then read that file first next time.
11Restart the conversation instead of stacking corrections (especially in Cowork)Since Cowork doesn't support editing earlier prompts, use "Restart the conversation from here" — the earlier the restart point, the more tokens saved.
12Summarize and start fresh every 15–20 messagesAsk Claude to summarize everything important, then paste that summary as the first message of a new session.
13Use Sonnet/Haiku for simple tasks, Opus for deep workRule of thumb: if a task would take under ~30 seconds to answer, it probably doesn't need Opus.
14Don't dump the entire folder into Cowork "just in case"Every file read costs tokens; overly large file sets may cause looser summarization instead of careful reading. For tasks with no file needs, select zero folders.
15Start a new chat when the topic changesOld, unrelated messages are still re-read as "dead weight" context on every new question.
16Turn off unused features per taskWeb search, connectors, "Explore" mode and Extended Thinking all add tokens to every response even when not needed. With connectors, specific queries (e.g., a defined time range and topic) are cheaper than broad, open-ended ones.
17Use Projects for recurring workUploading the same file to multiple chats re-tokenizes it every time. In a Project, it's uploaded once and reused; on paid plans, Projects use RAG to retrieve only relevant chunks instead of loading the entire document.
18Turn off Memory, use Personal Preferences/Styles insteadWithout saved context, new chats can waste several messages re-establishing tone/role/format. Personal Preferences and Styles (e.g., "Concise") persist across chats without adding to context every time.
19Use scheduled/recurring tasks (/schedule)For repeated reports/digests, instead of repeating them manually inside a growing session.
20Give Claude Code a clear scopeWithout a specific target, Code tends to explore broadly (directories, files, checks) — costing tokens. A precise instruction (exact target, input file, output filename) leaves little room for open-ended exploration.
21Use a CLAUDE.md file for permanent context in CodeCode reads CLAUDE.md before every task, similar to Cowork's "Global Instructions." Keep it short — occasional workflows belong in Skills instead, since Skills load on demand while CLAUDE.md loads every time.
22Spread work across the dayUsage limits run on a rolling 5-hour window. Burning the full limit in one morning session leaves capacity unused. Splitting into 2–3 sessions lets earlier usage roll off.
23Don't use Claude for tasks it isn't suited forExample: Claude cannot generate images — repeatedly trying to describe a visual wastes messages on a task it won't solve.

4. Where to Start

A sensible way to prioritize, depending on your usage pattern:

  • Daily Cowork users: habits 1, 2, 5 — convert files before upload, plan in Chat before building, stop asking for full redos.
  • Mostly Chat users: habits 8, 15, 17 — edit instead of correcting, new chat per topic, use Projects for recurring files.
  • Frequently hitting limits (lower-tier plan): habits 6, 13, 22 — batch prompts, use cheaper models for simple tasks, spread sessions across the day.
Context

These habits come from practitioner experience shared online, not an official Anthropic document. Some specifics (e.g., caching mechanisms or exactly how usage limits are weighted) are described as "confirmed by Anthropic" in the source but not published in full detail. When in doubt, check current limits and features in your own account or in Anthropic's official documentation.
