Why Claude is relevant in investment banking

Research, document analysis, drafting communications and structuring information are part of every analyst's and intern's daily work. Claude accelerates exactly these tasks — provided you know the features, formulate precise instructions and understand the boundaries. This course covers both: the practical value and the safe, responsible use.

Recommendation: work through the modules in order and try each exercise directly in Claude. The learning effect comes from doing, not just reading.

1. The Model Family: The Right Tool for Every Task

Claude is far more than a simple chatbot — it is a versatile ecosystem with specialised tools for productivity and development. The starting point is the model family:

ModelStrengthTypical use

Adaptive thinking — what's behind it?

Sonnet 5, Opus 4.8 and Fable 5 automatically decide, with no input from you, how much time to spend on an answer: for a simple question they reply instantly, for a complex analysis they think longer in the background — no manual toggle needed. Only Haiku 4.5 deliberately skips this to keep maximum speed.

Looking ahead: What's next?

Anthropic is continuously developing new models. Since June 2026, a new tier above Opus has existed — the "Mythos" class: Claude Fable 5 is its publicly available model, while the more capable Claude Mythos 5 is so far only available to select organisations as part of "Project Glasswing". For everyday work, Haiku, Sonnet and Opus remain the relevant tools — Fable is worthwhile only for rare exceptional cases with the highest requirements.

2. Organisation through "Projects" — The Briefing Folder

A Project is an outsourced brain with documents, instructions and context. Claude accesses it in every conversation within the project — ideal for recurring tasks:

  • Create a project → upload background documents (e.g. company profiles, style guides)
  • Define a system prompt: what role should Claude take within the project?
  • From that point on, every new chat in the project starts with this context — no re-briefing needed

3. Interactive Visualisation in Chat

Claude can generate interactive content directly in the chat — no plugins, no coding knowledge required:

  • Charts and dashboards: Interactive graphics, flowcharts and KPI dashboards
  • Calculation tools: Functional mini-apps with input fields and computed outputs
  • Simple command: "Show me this as an interactive chart" — Claude generates the visualisation immediately
Practical tip

For client presentations: Claude visualisations can be shown directly in the browser or captured as a screenshot for PowerPoint.

4. Claude Cowork — Desktop Assistant

Claude Cowork lives in the desktop app and carries out tasks autonomously on your computer:

  • Access to local folders and files — no manual uploading required
  • Works in the background while you focus on other tasks
  • Remote control via the smartphone app: delegate tasks even when you're away from your desk

5. Claude Code — Development Without Coding Knowledge

Claude Code makes technical implementation accessible to everyone:

  • A simple description is enough — Claude creates complete websites, tools or apps
  • No programming experience required: describe your requirements in plain language, code is generated automatically
  • Iterative improvement: give feedback and Claude adjusts the code accordingly

6. Extensions: Skills, Plugins and Connectors

The Claude ecosystem can be extended through three types of add-ons:

  • Skills: Store individual capabilities permanently — e.g. "always produce an Excel file as output"
  • Plugins: Collections of skills bundled as a package (e.g. a productivity plugin with writing and analysis functions)
  • Connectors: Direct interfaces to external services — Gmail, Google Calendar, Notion and more
Security notice

Connectors give Claude access to real data from external services. Never connect personal company accounts without approval — follow IEG's data protection guidelines.

Exercise 1

The right model for the right task

Haiku, Sonnet, Opus — or, in exceptional cases, Fable? Each model has clear strengths. Practice choosing the optimal model for different scenarios — and explain why.

  • Create a table with three columns: Task, Model and Rationale.
  • Evaluate these scenarios: (a) Quick formatting of a contact list, (b) In-depth analysis of an 80-page market study, (c) Summarising daily industry news, (d) Complex company valuation with multiple scenarios, (e) Creating an interactive KPI calculator.
  • Justify each decision in one sentence.
Example prompt:

I will give you 5 tasks from everyday work. Recommend the appropriate Claude model (Haiku, Sonnet, Opus, or Fable in exceptional cases) for each and justify the choice in one sentence. Format: table with three columns.

Goal: The right tool for the right purpose — think model and requirement together.

Exercise 2

Setting up your first Project

Projects are the heart of productivity in Claude. In this exercise, you set up your first Project for a typical IEG use case.

  • Open Claude and create a new Project — e.g. "M&A Research Assistant".
  • Define a system prompt: what role should Claude play in the project? (e.g. "You are an experienced M&A analyst. Always respond in English, structured and precise.")
  • Test the Project: ask a question and check whether Claude follows the defined context.
  • Optionally upload a short background document (e.g. a one-page company overview) to the Project — then ask about it in the chat.
Example system prompt for the Project:

You are an experienced M&A analyst at IEG. Always respond in English. Structure answers with clear sections. When uncertain, say so explicitly and recommend manual verification.

Goal: A well-configured Project saves time every day — no re-briefing, immediate start on every task.
Approach
  • Create Project: Left sidebar → "New Project" → assign a title (e.g. "Deal Research").
  • System prompt: Project settings → Instructions → define role, language and tone.
  • Test: Start a new chat in the Project → ask a question → Claude should respond directly in the defined style, without re-briefing.
  • Upload document: Paperclip icon → upload file → available in all Project chats from now on.

Conclusion: A well-configured Project is like a permanently available, fully briefed assistant — set up once, useful every day.
