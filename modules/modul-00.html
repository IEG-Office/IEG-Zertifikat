Das Claude-Ökosystem im Überblick

Modellfamilie, Projects, interaktive Visualisierungen, Claude Cowork, Claude Code und Erweiterungen — ein strukturierter Überblick über das gesamte Claude-Ökosystem als Einstieg in den Kurs.

25 Min. Lernzeit 10 Quiz-Fragen Mit Video

So nutzen Sie diesen Kurs — und warum Claude bei IEG zählt

Dieser Kurs führt Sie in sieben aufeinander aufbauenden Modulen vom ersten Chat bis zu professionellen Arbeitsabläufen. Jedes Modul kombiniert eine kurze Lerneinheit mit Praxisübungen und einem abschließenden Quiz. Erst wenn Sie ein Quiz mit mindestens 70 % bestehen, wird das nächste Modul freigeschaltet. Am Ende steht eine Abschlussprüfung über alle Inhalte — danach erhalten Sie Ihr persönliches Teilnahmezertifikat.

Warum Claude im Investment Banking relevant ist

Recherche, Dokumentenanalyse, Entwürfe für die Kommunikation und die Strukturierung von Informationen gehören zum Tagesgeschäft jedes Analysts und Interns. Claude beschleunigt genau diese Aufgaben — vorausgesetzt, man kennt die Funktionen, formuliert präzise Anweisungen und versteht die Einsatzgrenzen. Dieser Kurs vermittelt beides: den praktischen Nutzen und den sicheren, verantwortungsvollen Umgang.

Empfehlung: Arbeiten Sie die Module der Reihe nach durch und probieren Sie jede Übung direkt in Claude aus. Der Lerneffekt entsteht durch das eigene Ausprobieren, nicht durch reines Lesen.

1. Die Modellfamilie: Für jede Aufgabe das richtige Werkzeug

Claude ist weit mehr als ein einfacher Chatbot — es ist ein vielseitiges Ökosystem mit spezialisierten Werkzeugen für Produktivität und Entwicklung. Den Einstieg bildet die Modellfamilie:

ModellStärkeTypischer Einsatz
HaikuDas schnellste und kostengünstigste ModellEinfache Aufgaben, schnelle Formatierungen, kurze Zusammenfassungen
SonnetDer „Allrounder"Ausgewogenes Verhältnis aus Geschwindigkeit und Intelligenz — tägliche Arbeit
OpusDas Flaggschiff-ModellMaximale Leistungsfähigkeit für extrem komplexe Analysen und Entscheidungen
Ausblick: Was kommt als Nächstes?

Anthropic arbeitet kontinuierlich an neuen Modellen. Der aktuelle Entwicklungsstand zeigt, wohin die Reise geht — Geschwindigkeit, Tiefe und Integration werden weiter zunehmen.

2. Organisation durch „Projects" — Der Briefing-Ordner

Ein Project ist ein ausgelagertes Gehirn mit Dokumenten, Anweisungen und Kontext. Claude greift bei jedem Gespräch innerhalb des Projekts darauf zu — ideal für wiederkehrende Aufgaben:

  • Projekt anlegen → Hintergrunddokumente hochladen (z. B. Unternehmensprofile, Styleguides)
  • Systemprompt definieren: Welche Rolle soll Claude im Projekt einnehmen?
  • Ab sofort startet jeder neue Chat im Projekt mit diesem Kontext — kein erneutes Briefing nötig

3. Interaktive Visualisierung im Chat

Claude kann direkt im Chat interaktive Inhalte erzeugen — ohne Plugins, ohne Programmierkenntnisse:

  • Diagramme und Dashboards: Interaktive Grafiken, Flowcharts und Kennzahlen-Dashboards
  • Bewertungsrechner: Funktionale Mini-Apps mit Eingabefeldern und Berechnungen
  • Einfacher Befehl: „Zeig mir das als interaktives Diagramm" — Claude generiert die Visualisierung sofort
Praxis-Tipp

Für Kundenpräsentationen: Claude-Visualisierungen können direkt im Browser gezeigt oder als Screenshot für PowerPoint verwendet werden.

4. Claude Cowork — Desktop-Assistent

Claude Cowork lebt in der Desktop-App und arbeitet Aufgaben selbstständig auf dem Computer ab:

  • Zugriff auf lokale Ordner und Dateien — ohne manuelles Hochladen
  • Arbeitet auch im Hintergrund, während andere Tätigkeiten erledigt werden
  • Fernsteuerung über die Smartphone-App: Aufgaben delegieren, auch wenn man nicht am Rechner sitzt

5. Claude Code — Entwicklung ohne Programmierkenntnisse

Claude Code macht technische Realisierung für jeden zugänglich:

  • Einfache Beschreibung reicht — Claude erstellt vollständige Webseiten, Tools oder Apps
  • Kein Programmier-Vorwissen notwendig: Anforderungen auf Deutsch formulieren, Code wird automatisch generiert
  • Iterative Verbesserung: Feedback geben, Claude passt den Code an

6. Erweiterungen: Skills, Plugins und Konnektoren

Das Claude-Ökosystem lässt sich durch drei Typen von Erweiterungen ausbauen:

  • Skills: Einzelne Fähigkeiten dauerhaft speichern — z. B. „Erstelle immer eine Excel-Datei als Ausgabe"
  • Plugins: Sammlungen von Skills gebündelt als Paket (z. B. ein Productivity-Plugin mit Schreib- und Analysefunktionen)
  • Konnektoren: Direkte Schnittstellen zu externen Diensten — Gmail, Google Calendar, Notion und weitere
Sicherheitshinweis

Konnektoren geben Claude Zugriff auf echte Daten externer Dienste. Niemals persönliche Firmen-Accounts ohne Freigabe verbinden — Datenschutzrichtlinien von IEG beachten.

Praxisübung

Das Gelernte anwenden

Erkunden Sie das Claude-Ökosystem aktiv: Modellwahl, Projects und die ersten Erweiterungen.

Übung 1

Das richtige Modell für die richtige Aufgabe

Haiku, Sonnet oder Opus? Jedes Modell hat klare Stärken. Trainieren Sie, für verschiedene Szenarien das optimale Modell zu wählen — und warum.

  • Erstellen Sie eine Tabelle mit drei Spalten: Aufgabe, Modell und Begründung.
  • Bewerten Sie diese Szenarien: (a) Schnelle Formatierung einer Kontaktliste, (b) Tiefgehende Analyse einer 80-seitigen Marktstudie, (c) Tagesaktuelle Branchennews zusammenfassen, (d) Komplexe Unternehmensbewertung mit mehreren Szenarien, (e) Interaktiven Kennzahlen-Rechner erstellen.
  • Begründen Sie jede Entscheidung in einem Satz.
Beispiel-Prompt:

Ich gebe dir 5 Aufgaben aus dem Arbeitsalltag. Empfehle jeweils das passende Claude-Modell (Haiku, Sonnet, Opus) und begründe die Wahl in einem Satz. Format: Tabelle mit drei Spalten.

Ziel: Das richtige Werkzeug für den richtigen Zweck — Modell und Anforderung zusammendenken.

Übung 2

Ein erstes Project einrichten

Projects sind das Herzstück der Produktivität in Claude. In dieser Übung richten Sie Ihr erstes Project für einen typischen IEG-Anwendungsfall ein.

  • Öffnen Sie Claude und erstellen Sie ein neues Project — z. B. „M&A Research Assistant".
  • Definieren Sie einen Systemprompt: Welche Rolle soll Claude im Projekt einnehmen? (z. B. „Du bist ein erfahrener M&A-Analyst. Antworte immer auf Deutsch, strukturiert und präzise.")
  • Testen Sie das Project: Stellen Sie eine Frage und prüfen Sie, ob Claude den definierten Kontext einhält.
  • Laden Sie optional ein kurzes Hintergrunddokument (z. B. eine Seite Unternehmensübersicht) in das Project — und fragen Sie danach im Chat.
Beispiel-Systemprompt für das Project:

Du bist ein erfahrener M&A-Analyst bei IEG. Antworte immer auf Deutsch. Strukturiere Antworten mit klaren Abschnitten. Bei Unsicherheiten weise explizit darauf hin und empfehle eine manuelle Überprüfung.

Ziel: Ein einmal eingerichtetes Project spart täglich Zeit — kein erneutes Briefing, sofortiger Einstieg in jede Aufgabe.
Lösungsansatz
  • Project anlegen: Linke Seitenleiste → „Neues Projekt" → Titel vergeben (z. B. „Deal Research").
  • Systemprompt: Projekteinstellungen → Instruktionen → Rolle, Sprache und Ton definieren.
  • Test: Neuen Chat im Project starten → Frage stellen → Claude sollte direkt im definierten Stil antworten, ohne erneutes Briefing.
  • Dokument hochladen: Paperclip-Symbol → Datei hochladen → ab sofort in allen Chats des Projects verfügbar.

Fazit: Ein gut eingerichtetes Project ist wie ein permanent verfügbarer, vollständig gebriefter Assistent — einmal angelegt, täglich nützlich.
